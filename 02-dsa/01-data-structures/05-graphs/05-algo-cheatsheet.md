# 05. Graph Algorithms (Cheat Sheet & Use Cases)

গ্রাফের অনেকগুলো পাওয়ারফুল অ্যালগরিদম রয়েছে। ইন্টারভিউতে সবচেয়ে বেশি জিজ্ঞেস করা হয়—"কোন অ্যালগরিদম কখন ব্যবহার করবো?" এই ফাইলে প্রতিটি অ্যালগরিদমের Pros, Cons, Real-world Use Cases এবং Tricky পয়েন্টগুলো আলোচনা করা হলো।

---

## 1. Shortest Path Algorithms (শর্টেস্ট পাথ)

### A. Dijkstra's Algorithm
- **কাজ:** সিঙ্গল সোর্স (Single Source) থেকে বাকি সব নোডের সবচেয়ে ছোট রাস্তা বের করা।
- **Pros:** অত্যন্ত ফাস্ট। টাইম কমপ্লেক্সিটি $O(E \log V)$ (Priority Queue ব্যবহার করলে)।
- **Cons:** নেগেটিভ ওয়েট (Negative Weight) থাকলে কাজ করে না। ইনফিনিট লুপে পড়ে যেতে পারে বা ভুল উত্তর দিতে পারে।
- **Real-world Use Case:** গুগল ম্যাপস বা উবারের রুট নেভিগেশন (কারণ রাস্তার দূরত্ব কখনো নেগেটিভ হয় না), নেটওয়ার্ক রাউটিং প্রোটোকল (OSPF)।
- **Trick:** ইন্টারভিউতে যদি বলে "Unweighted Graph-এ শর্টেস্ট পাথ বের করো", তখন Dijkstra এর বদলে **BFS** ব্যবহার করবেন (BFS এর কমপ্লেক্সিটি $O(V+E)$, যা Dijkstra এর চেয়ে ফাস্ট)।

### B. Bellman-Ford Algorithm
- **কাজ:** সিঙ্গল সোর্স শর্টেস্ট পাথ বের করা, বিশেষ করে নেগেটিভ ওয়েট থাকলে।
- **Pros:** নেগেটিভ ওয়েট এজ (Negative Weight Edges) হ্যান্ডেল করতে পারে। এটি "Negative Weight Cycle" ডিটেক্ট করতে পারে।
- **Cons:** অনেক স্লো। টাইম কমপ্লেক্সিটি $O(V \times E)$। 
- **Real-world Use Case:** ফিনান্সিয়াল আর্বিট্রেজ (Financial Arbitrage) বা কারেন্সি এক্সচেঞ্জে প্রফিট লুপ (Negative Cycle) বের করতে ব্যবহৃত হয়।

### C. Floyd-Warshall Algorithm
- **কাজ:** All-Pairs Shortest Path (যেকোনো নোড থেকে যেকোনো নোডের শর্টেস্ট পাথ)।
- **Pros:** কোড করা খুবই সহজ (মাত্র ৩টা Nested Loop)। নেগেটিভ ওয়েট হ্যান্ডেল করতে পারে।
- **Cons:** স্পেস এবং টাইম কমপ্লেক্সিটি প্রচুর—$O(V^3)$। তাই নোড সংখ্যা ৪০০-৫০০ এর বেশি হলে এটি অকেজো।
- **Real-world Use Case:** ছোট বা ফিক্সড সাইজের নেটওয়ার্কে (যেমন একটি বিল্ডিংয়ের সব রুমের দূরত্ব) ম্যাপ তৈরি করা।

### D. A* (A-Star) Search Algorithm
- **কাজ:** সোর্স থেকে একটি নির্দিষ্ট ডেস্টিনেশনের শর্টেস্ট পাথ বের করা (Dijkstra এর স্মার্ট ভার্সন)।
- **Pros:** এটি একটি হিউরিস্টিক (Heuristic) ফাংশন ব্যবহার করে অনুমান করে কোন দিকে গেলে তাড়াতাড়ি পৌঁছানো যাবে, তাই এটি Dijkstra এর চেয়ে অনেক দ্রুত ডেস্টিনেশন খুঁজে পায়।
- **Cons:** হিউরিস্টিক ফাংশন ঠিকমতো ডিজাইন না করলে এটি ভুল পাথ দিতে পারে।
- **Real-world Use Case:** গেম ডেভেলপমেন্টে NPC (Non-Player Characters) এর পাথফাইন্ডিং, রোবোটিক্স।

## 2. Minimum Spanning Tree (MST)
MST হলো এমন একটি সাব-ট্রি যা গ্রাফের সব নোডকে কানেক্ট করে, কিন্তু টোটাল এজ ওয়েট সবচেয়ে কম হয়।

### E. Kruskal's Algorithm
- **কাজ:** গ্রাফ থেকে MST বের করা।
- **Pros:** এজগুলোকে সর্ট করে ছোট থেকে বড় অনুযায়ী কাজ করে। Sparse Graph (এজ কম থাকলে) এর জন্য এটি চমৎকার।
- **Cons:** এজগুলো সর্ট করতে $O(E \log E)$ সময় লাগে।
- **Real-world Use Case:** নেটওয়ার্ক বা পাওয়ার গ্রিড ডিজাইন করা (যাতে সবচেয়ে কম তার ব্যবহার করে সব শহরে বিদ্যুৎ দেওয়া যায়)।
- **Key Concept:** এটি ইমপ্লিমেন্ট করতে **Union-Find (Disjoint Set)** ডেটা স্ট্রাকচার লাগে।

### F. Prim's Algorithm
- **কাজ:** MST বের করা।
- **Pros:** একটি নোড থেকে শুরু করে আস্তে আস্তে ট্রি বড় করে। Dense Graph (যেখানে অনেক কানেকশন) এর জন্য এটি Kruskal এর চেয়ে ফাস্ট।
- **Cons:** Priority Queue মেইনটেইন করতে হয়।
- **Real-world Use Case:** ল্যান (LAN) নেটওয়ার্ক ডিজাইন।

## 3. Graph Connectivity & Cycles

### G. Union-Find (Disjoint Set)
- **কাজ:** কোনো গ্রাফে সাইকেল আছে কি না তা বের করা এবং কানেক্টেড কম্পোনেন্ট বের করা।
- **Pros:** Path Compression এবং Union by Rank ব্যবহার করলে এর অপারেশনগুলো প্রায় $O(1)$ টাইমে কাজ করে ($O(\alpha(V))$)।
- **Cons:** শুধুমাত্র Undirected গ্রাফে সাইকেল বের করতে পারে। Directed গ্রাফে কাজ করে না।
- **Use Case:** ইমেজে অবজেক্ট আইডেন্টিফাই করা, নেটওয়ার্কে কানেক্টিভিটি চেক করা।

### H. Tarjan's / Kosaraju's Algorithm
- **কাজ:** Strongly Connected Components (SCC) বের করা (Directed Graph এর ক্ষেত্রে)। SCC মানে এমন একটি অংশ যেখানে যেকোনো নোড থেকে যেকোনো নোডে যাওয়া যায়।
- **Pros:** টাইম কমপ্লেক্সিটি $O(V+E)$।
- **Real-world Use Case:** সোশ্যাল নেটওয়ার্কে কমিউনিটি (Community/Clusture) বের করা, গুগল পেজ-র‍্যাঙ্ক অপটিমাইজ করা।

## 4. Directed Acyclic Graph (DAG)

### I. Topological Sorting
- **কাজ:** DAG এর নোডগুলোকে একটি লিনিয়ার অর্ডারে সাজানো, যাতে প্রতিটি এজ $U \to V$ এর জন্য $U$ সবসময় $V$ এর আগে থাকে।
- **Pros:** টাইম কমপ্লেক্সিটি $O(V+E)$।
- **Cons:** সাইক্লিক গ্রাফে কাজ করে না।
- **Real-world Use Case:** 
  - **কোর্স প্রিরিকুইজিট:** CS101 না করে CS201 করা যাবে না।
  - **বিল্ড সিস্টেম (Build Systems):** NPM, Maven বা Gradle-এ কোন প্যাকেজ আগে ইনস্টল করতে হবে তা বের করা।
  - **টাস্ক শিডিউলিং:** প্রজেক্ট ম্যানেজমেন্ট সফটওয়্যারে ডিপেন্ডেন্সি ম্যানেজ করা।

## Quick Summary (ইন্টারভিউয়ের জন্য)

| প্রবলেম / সিনারিও | কোন অ্যালগরিদম ব্যবহার করবেন? |
| :--- | :--- |
| Unweighted গ্রাফে Shortest Path | **BFS** |
| Positive Weighted গ্রাফে Shortest Path | **Dijkstra's** |
| Negative Edge থাকলে Shortest Path | **Bellman-Ford** |
| সব জোড়া (All-pairs) Shortest Path | **Floyd-Warshall** |
| সবচেয়ে কম খরচে সব নোড কানেক্ট করা | **Kruskal (Sparse) বা Prim (Dense)** |
| কাজের সিরিয়াল বা ডিপেন্ডেন্সি বের করা | **Topological Sort** |
| সাইকেল ডিটেক্ট করা (Undirected) | **Union-Find বা DFS** |
| গেমের ক্যারেক্টার নেভিগেশন | **A* (A-Star)** |
