# Java String Concepts

## `==` বনাম `.equals()` এবং String Pool

জাভাতে `==` অপারেটর অবজেক্টের ক্ষেত্রে (যেমন String) **শুধুমাত্র মেমরি অ্যাড্রেস (Reference)** চেক করে। অর্থাৎ, এটি দেখে দুটি ভেরিয়েবল মেমরির ঠিক একই জায়গাকে নির্দেশ করছে কি না। এর মান (value) বা টাইপ কী, তা `==` অপারেটর দেখে না।

**উদাহরণ:**
```java
String a = "Test"; 
String c = new String("Test");
System.out.println(a == c); // আউটপুট: false
```

*   `String a = "Test";` - এটি Heap মেমরির String Pool-এ মেমরি নেয়।
*   `String c = new String("Test");` - `new` কীওয়ার্ড ব্যবহার করার কারণে এটি সম্পূর্ণ নতুন করে Heap মেমরিতে আরেকটি অবজেক্ট তৈরি করে (String Pool এর বাইরে)।

যেহেতু `a` এবং `c` এর মেমরি অ্যাড্রেস আলাদা, তাই `a == c` এর ফলাফল `false` হয়। মেমরি অ্যাড্রেস না দেখে **শুধুমাত্র মান (Value)** চেক করতে চাইলে `.equals()` মেথড ব্যবহার করতে হবে: `a.equals(c)`.

**একইভাবে ডিক্লেয়ার করলে:**
```java
String a = "Test";
String c = "Test";
System.out.println(a == c); // আউটপুট: true
```
জাভা মেমরি বাঁচানোর জন্য নতুন কোনো অবজেক্ট তৈরি না করে String Pool থেকে একই অবজেক্ট রিইউজ করে, তাই `a` এবং `c` একই জায়গায় পয়েন্ট করে।

## StringBuilder বনাম StringBuffer

জাভাতে `StringBuilder` এবং `StringBuffer` দুটি ক্লাসই **Mutable String** (যে স্ট্রিংয়ের মান পরিবর্তন করা যায়) তৈরি করার জন্য ব্যবহৃত হয়। 

### মূল পার্থক্য:

| বৈশিষ্ট্য | `StringBuffer` | `StringBuilder` |
| :--- | :--- | :--- |
| **Thread Safety** | Thread-safe (একই সাথে একাধিক থ্রেড অ্যাক্সেস করতে পারে না)। | Not Thread-safe (একাধিক থ্রেড একই সাথে অ্যাক্সেস করতে পারে)। |
| **Synchronization** | এর মেথডগুলো Synchronized। | এর মেথডগুলো Synchronized নয়। |
| **Performance** | তুলনামূলকভাবে **ধীর (Slow)**। কারণ লকিং মেকানিজম ব্যবহার করে। | তুলনামূলকভাবে **অনেক দ্রুত (Fast)**। কারণ লকিংয়ের ঝামেলা নেই। |
| **জাভা ভার্সন** | জাভার শুরু থেকেই (Java 1.0) আছে। | জাভা ১.৫ (Java 1.5) ভার্সনে এটি যুক্ত করা হয়। |

### কখন কোনটি ব্যবহার করবেন?
*   **StringBuilder:** যখন আপনি সিঙ্গেল-থ্রেড (Single-thread) অ্যাপ্লিকেশন নিয়ে কাজ করছেন। সাধারণ প্রোগ্রামিংয়ে এটিই সবচেয়ে বেশি ব্যবহৃত হয় কারণ এটি দ্রুত কাজ করে।
*   **StringBuffer:** শুধুমাত্র মাল্টি-থ্রেডেড (Multi-threaded) পরিবেশে, যেখানে থ্রেড-সেফটি নিশ্চিত করা প্রয়োজন।
