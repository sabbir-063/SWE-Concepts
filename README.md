# Software Engineering (SWE) Learning Concepts

Welcome to the Software Engineering learning repository! This project is dedicated to learning and practicing core Software Engineering concepts step by step, including **Design Patterns**, **SOLID Principles**, and **System Architectures**.

As new concepts are learned, this repository will be continuously updated with new implementations and notes.

## 📂 Current Contents

### 1. Design Patterns (`/DesignPatterns`)
Contains Java implementations of the Gang of Four (GoF) design patterns, along with a comprehensive bilingual (Bengali/English) guide (`README.md` inside the folder). The patterns are categorized into three main areas:
- **Behavioral/**: Patterns that identify common communication patterns between objects.
- **Creational/**: Patterns that deal with object creation mechanisms.
- **Structural/**: Patterns that ease the design by identifying a simple way to realize relationships between entities.

### 2. SOLID Principles (`/SOLID`)
A complete, step-by-step guide to mastering the SOLID principles using Java. Each principle has its own folder containing a bad example (violating the principle), a good example (refactored), an industry-level example, and a detailed Bengali/English `readme.md` with instructions on how to run them.
- **SRP/**: Single Responsibility Principle
- **OCP/**: Open/Closed Principle
- **LSP/**: Liskov Substitution Principle
- **ISP/**: Interface Segregation Principle
- **DIP/**: Dependency Inversion Principle
- **GrandFinale/**: A mini-project refactoring a chaotic E-Commerce God Class by applying all 5 SOLID principles simultaneously.
