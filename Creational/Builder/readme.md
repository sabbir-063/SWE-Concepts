# 4. Builder Pattern (বিল্ডার প্যাটার্ন)

### 📖 মূল কনসেপ্ট (Definition)
**সহজ বাংলায়:** যখন কোনো অবজেক্ট তৈরি করার প্রক্রিয়াটি অনেক জটিল হয় (অর্থাৎ কনস্ট্রাক্টরে অনেকগুলো প্যারামিটার পাস করতে হয়), তখন আমরা বিল্ডার প্যাটার্ন ব্যবহার করি। এটি ধাপে ধাপে (Step-by-step) অবজেক্ট তৈরি করতে সাহায্য করে এবং কোডকে অনেক বেশি রিডেবল (Readable) করে তোলে।

**In English:** Builder pattern lets you construct complex objects step by step. The pattern allows you to produce different types and representations of an object using the same construction code.

---

### 💻 কিভাবে কাজ করে? (Implementation Details)
1. অবজেক্টের ক্লাসের ভেতরেই একটি `static nested Builder` ক্লাস তৈরি করা হয়।
2. বিল্ডার ক্লাসে আসল ক্লাসের সব ভ্যারিয়েবল থাকে।
3. বিল্ডার ক্লাসে প্রতিটি ভ্যারিয়েবল সেট করার জন্য আলাদা মেথড থাকে (যেমন: `setCheese()`, `setPepperoni()`) যেগুলো আবার বিল্ডার অবজেক্টকেই রিটার্ন করে (এটিকে Method Chaining বলে)।
4. সবশেষে একটি `build()` মেথড থাকে যা আসল অবজেক্টটি তৈরি করে রিটার্ন করে।

---

### 🧪 বাস্তব জীবনের উদাহরণ (Industry Use Cases)
1. **Lombok `@Builder`:** জাভাতে লম্বক লাইব্রেরির `@Builder` অ্যানোটেশনটি এই প্যাটার্নেরই একটি ডায়নামিক ইমপ্লিমেন্টেশন।
2. **Query Builders:** ডাটাবেস কোয়েরি বিল্ড করার সময় (`SELECT * FROM table WHERE ... ORDER BY ... LIMIT ...`) বিল্ডার প্যাটার্ন ব্যবহার করা হয়।
3. **Food Ordering (Pizza/Burger):** একটি পিজ্জা বানানোর সময় ধাপে ধাপে এক্সট্রা চিজ, সস বা অলিভ অ্যাড করার জন্য এটি দারুণ কাজ করে।

---

### ✅ সুবিধা (Pros)
- টেলিস্কোপিং কনস্ট্রাক্টরের (Telescoping constructor - অনেক প্যারামিটারওয়ালা কনস্ট্রাক্টর) সমস্যা দূর করে।
- ধাপে ধাপে অবজেক্ট তৈরি করা যায়।
- কোড পড়া এবং বোঝা অনেক সহজ হয়ে যায় (Fluent API)।

### ❌ অসুবিধা (Cons)
- ক্লাসের ভেতরে একটি এক্সট্রা বিল্ডার ক্লাস লিখতে হয়, তাই কোডের দৈর্ঘ্য বেড়ে যায়।
