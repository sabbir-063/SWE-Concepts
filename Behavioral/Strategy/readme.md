# 20. Strategy Pattern (স্ট্র্যাটেজি প্যাটার্ন)

### 📖 মূল কনসেপ্ট (Definition)
**সহজ বাংলায়:** আপনার কাছে যদি একটি কাজ করার অনেকগুলো আলাদা আলাদা উপায় বা অ্যালগরিদম (Algorithm) থাকে, তবে প্রতিটা উপায়কে আলাদা ক্লাসে (Strategy) রাখা হয়। এতে করে ক্লায়েন্ট তার পছন্দমতো যেকোনো একটি উপায় বেছে নিতে পারে এবং রান-টাইমে (Run-time) চাইলে একটি উপায় বদলে অন্যটি ব্যবহার করতে পারে। 

ধরুন, আপনি গুগল ম্যাপসে (Google Maps) এক জায়গা থেকে অন্য জায়গায় যাবেন। গুগল আপনাকে তিনটি অপশন দেয়: ১. গাড়িতে যাওয়া, ২. বাইকে যাওয়া, ৩. হেঁটে যাওয়া। গন্তব্য কিন্তু একটাই, শুধু যাওয়ার পথ বা "স্ট্র্যাটেজি" আলাদা। আপনি যেকোনো মুহূর্তে একটি স্ট্র্যাটেজি সিলেক্ট করতে পারেন। এটাই হলো স্ট্র্যাটেজি প্যাটার্ন!

**In English:** Strategy pattern defines a family of algorithms, encapsulates each one, and makes them interchangeable. It lets the algorithm vary independently from the clients that use it.

---

### 🧩 প্যাটার্নের কম্পোনেন্টগুলো (Components of the Pattern)
এই প্যাটার্নে ৩টি মূল কম্পোনেন্ট থাকে:

1. **Context (কনটেক্সট):** এটি হলো আমাদের আসল অবজেক্ট (যেমন: `ShoppingCart` বা `GoogleMaps`)। এর ভেতরে বর্তমান স্ট্র্যাটেজির একটি রেফারেন্স রাখা থাকে। কনটেক্সট নিজে কোনো কাজ করে না, সে কাজটা স্ট্র্যাটেজিকে করতে দেয়।
2. **Strategy (ইন্টারফেস):** এটি সব অ্যালগরিদমের জন্য কমন একটি রুলস বা মেথড (যেমন: `pay()`, `buildRoute()`) ডিক্লেয়ার করে।
3. **Concrete Strategies (আসল অ্যালগরিদমগুলো):** এগুলো হলো আলাদা আলাদা অ্যালগরিদম বা উপায় (যেমন: `BkashPayment`, `CreditCardPayment`)।

---

### 💻 কিভাবে কাজ করে? (Java Implementation)
আপনার ফোল্ডারে থাকা `Main.java` ফাইলটি ওপেন করুন। সেখানে ই-কমার্স সাইটের পেমেন্ট সিস্টেমের (Payment Gateway) একটি দারুণ উদাহরণ দেওয়া আছে, যেখানে ইউজার রান-টাইমে বিকাশ বা ক্রেডিট কার্ড চুজ করতে পারে।

---

### 🔍 Deep Explanation (কেন আমরা if-else ব্যবহার করবো না?)
যদি আমরা স্ট্র্যাটেজি প্যাটার্ন ব্যবহার না করতাম, তবে `ShoppingCart` ক্লাসের ভেতরে পেমেন্ট লজিক দেখতে হতো এরকম:
```java
public void checkout(String method, int amount) {
    if (method.equals("bKash")) {
        // বিকাশের এপিআই কল করার ১০০ লাইনের কোড
    } else if (method.equals("CreditCard")) {
        // ক্রেডিট কার্ডের এপিআই কল করার ২০০ লাইনের কোড
    } else if (method.equals("CashOnDelivery")) {
        // ...
    }
}
```
এখানে সমস্যা হলো— প্রতিবার নতুন একটি পেমেন্ট গেটওয়ে (যেমন: Nagad, PayPal) আসলে আপনাকে এই ক্লাসটি এডিট করতে হবে (Open/Closed Principle ভাঙবে)। ক্লাসটি হাজার লাইনের জগাখিচুড়ি হয়ে যাবে। 
স্ট্র্যাটেজি প্যাটার্ন এই বিশাল `if-else` ব্লকগুলোকে কেটে এক-একটি আলাদা ক্লাসে রূপান্তর করে দেয়।

---

### 🧪 বাস্তব জীবনের উদাহরণ (Industry Use Cases)
1. **Payment Methods / E-commerce:** আমরা `Main.java` তে যেই ইমপ্লিমেন্টেশনটা করেছি।
2. **Sorting Algorithms:** আপনার কাছে একটি লিস্ট আছে এবং আপনি চাইলে যেকোনো একটি সর্টিং অ্যালগরিদম (Bubble Sort, Merge Sort, Quick Sort) রান-টাইমে সিলেক্ট করতে পারেন।
3. **File Compression:** ফাইল সেভ করার সময় আপনি ঠিক করতে পারেন সেটা কোন ফরম্যাটে (ZIP, RAR, 7z) সেভ হবে।

---

### ✅ সুবিধা (Pros)
- রান-টাইমে অ্যালগরিদম সোয়াপ (Swap/পরিবর্তন) করা যায়।
- **Single Responsibility Principle:** প্রতিটি অ্যালগরিদমের নিজস্ব লজিক তার নিজস্ব ক্লাসেই থাকে। 
- নতুন অ্যালগরিদম তৈরি করা খুব সহজ, কনটেক্সট ক্লাসে হাত দিতে হয় না (Open/Closed Principle)।
- ক্লাসকে বিশাল `if-else` বা `switch` স্টেটমেন্টের হাত থেকে বাঁচায়।

### ❌ অসুবিধা (Cons)
- ক্লায়েন্টকে অবশ্যই জানতে হবে যে কোন স্ট্র্যাটেজিটা কীভাবে কাজ করে। কারণ ক্লায়েন্টকেই তো স্ট্র্যাটেজিটা সিলেক্ট করে দিতে হয়।
- যদি আপনার কাছে মাত্র দুই-তিনটি সাধারণ স্ট্র্যাটেজি থাকে যেগুলো খুব কমই বদলায়, তবে এই প্যাটার্নটি ওভার-ইঞ্জিনিয়ারিং হতে পারে।

### 💡 Java Interview Tips
- **Question:** "State Pattern এবং Strategy Pattern-এর মধ্যে আসল পার্থক্য কী?"
  **Answer:** 
  - **Strategy Pattern-এ** ক্লায়েন্ট নিজে কনটেক্সটকে বলে দেয় সে কোন স্ট্র্যাটেজি ব্যবহার করবে। স্ট্র্যাটেজিগুলো সাধারণত নিজেদের মধ্যে একে অপরকে চেনে না।
  - **State Pattern-এ** কনটেক্সট নিজেই নিজের স্টেট পরিবর্তন করে। স্টেটগুলো একে অপরকে চেনে এবং একটি স্টেট থেকে অন্য স্টেটে ট্রান্সফার (Transition) হতে পারে ক্লায়েন্টের সাহায্য ছাড়াই।
- **Question:** "Template Method Pattern এবং Strategy Pattern-এর মধ্যে পার্থক্য কী?"
  **Answer:** 
  - Strategy প্যাটার্নে ইনহেরিট্যান্সের (Inheritance) বদলে **Composition** ব্যবহার করা হয় এবং রান-টাইমে পাল্টানো যায়। 
  - Template Method-এ **Inheritance** ব্যবহার করা হয় (অ্যাবস্ট্রাক্ট ক্লাস) এবং এটা কম্পাইল টাইমে ফিক্সড থাকে।
