# Chapter 2: OSI & TCP/IP Reference Models

স্বাগতম! আমাদের কম্পিউটার নেটওয়ার্কিং কোর্সের ২য় অধ্যায়ে। এই অধ্যায়ে আমরা নেটওয়ার্কিংয়ের সবচেয়ে গুরুত্বপূর্ণ দুটি থিওরিটিক্যাল ফ্রেমওয়ার্ক—**OSI Model** এবং **TCP/IP Model** নিয়ে বিস্তারিত আলোচনা করব। 

---

## ১. Why Layered Models? (লেয়ারড মডেলের প্রয়োজন কেন?)
নেটওয়ার্কের মাধ্যমে একটি কম্পিউটার থেকে অন্য কম্পিউটারে ডেটা পাঠানোর প্রক্রিয়াটি অত্যন্ত জটিল। যদি এই পুরো কাজটি একটিমাত্র প্রোগ্রাম দিয়ে করা হতো, তবে কোনো সমস্যা হলে তা সমাধান করা বা নতুন কোনো প্রযুক্তি যোগ করা অসম্ভব হয়ে পড়ত।

তাই পুরো প্রক্রিয়াটিকে কয়েকটি স্তরে বা **Layers**-এ ভাগ করা হয়েছে। একে বলা হয় **Layered Architecture**। এর সুবিধাগুলো হলো:
1. **Interoperability**: বিভিন্ন কোম্পানির তৈরি হার্ডওয়্যার ও সফটওয়্যার যেন একসাথে কাজ করতে পারে।
2. **Easy Troubleshooting**: নেটওয়ার্কের কোন স্তরে সমস্যা হচ্ছে তা দ্রুত চিহ্নিত করা যায়।
3. **Modularity**: এক স্তরের প্রযুক্তি পরিবর্তন করলে অন্য স্তরে কোনো প্রভাব পড়ে না (যেমন: Wi-Fi থেকে Ethernet ক্যাবলে পরিবর্তন করলেও ব্রাউজারের HTTP প্রোটোকলে কোনো পরিবর্তন করতে হয় না)।

---

## ২. The OSI Reference Model (৭টি স্তর)
**OSI (Open Systems Interconnection)** মডেলটি ১৯৮৪ সালে **ISO** (International Organization for Standardization) দ্বারা তৈরি করা হয়। এটি একটি ৭-লেয়ার বিশিষ্ট তাত্ত্বিক (Theoretical) মডেল।

সহজে মনে রাখার উপায় (নিচ থেকে উপরে): 
> **P**lease **D**o **N**ot **T**hrow **S**ausage **P**izza **A**way
> (**P**hysical, **D**ata Link, **N**etwork, **T**ransport, **S**ession, **P**resentation, **A**pplication)

```
       +-----------------------+
       | 7. Application Layer  | <--- ইউজার ইন্টারফেস (HTTP, DNS)
       +-----------------------+
       | 6. Presentation Layer | <--- এনক্রিপশন, কম্প্রেশন, ফরম্যাটিং (JSON, JPEG)
       +-----------------------+
       | 5. Session Layer      | <--- সেশন তৈরি, নিয়ন্ত্রণ ও সমাপ্তি
       +-----------------------+
       | 4. Transport Layer    | <--- প্রসেস-টু-প্রসেস ডেলিভারি, পোর্ট নং (TCP/UDP)
       +-----------------------+
       | 3. Network Layer      | <--- হোস্ট-টু-হোস্ট ডেলিভারি, আইপি রাউটিং (IP)
       +-----------------------+
       | 2. Data Link Layer    | <--- হোপ-টু-হোপ ফিজিক্যাল ডেলিভারি, ম্যাক অ্যাড্রেস (MAC)
       +-----------------------+
       | 1. Physical Layer     | <--- ক্যাবল, কানেক্টর, বিট ট্রান্সমিশন
       +-----------------------+
```

### প্রতিটি লেয়ারের বিস্তারিত বিবরণ:

#### Layer 7: Application Layer (অ্যাপ্লিকেশন লেয়ার)
*   **কাজ**: এটি ইউজারের সবচেয়ে কাছের লেয়ার। ইউজার যেসব সফটওয়্যার বা ব্রাউজার ব্যবহার করে তার সাথে নেটওয়ার্ক সার্ভিসের সংযোগ ঘটায় এই লেয়ার।
*   **প্রধান প্রোটোকল**: HTTP/HTTPS (ওয়েব), SMTP/IMAP (ইমেইল), FTP (ফাইল ট্রান্সফার), DNS (ডোমেইন রেজোলিউশন)।

#### Layer 6: Presentation Layer (প্রেজেন্টেশন লেয়ার)
*   **কাজ**: একে "Syntax Layer"-ও বলা হয়। এটি ডেটার ফরম্যাটিং, ট্রান্সলেশন, কম্প্রেশন (সাইজ কমানো) এবং এনক্রিপশন/ডিক্রিপশন (নিরাপত্তা) নিশ্চিত করে।
*   **উদাহরণ**: ASCII, UTF-8, SSL/TLS, JPEG, MP4, JSON/XML ফরম্যাট।

#### Layer 5: Session Layer (সেশন লেয়ার)
*   **কাজ**: দুটি ডিভাইসের মধ্যকার যোগাযোগের সেশন (Session) তৈরি করা, তা বজায় রাখা এবং কাজ শেষে বন্ধ (Termination) করার কাজটি এই লেয়ার করে। এটি চেকপয়েন্ট বা সিনক্রোনাইজেশন পয়েন্ট তৈরি করে যাতে সংযোগ কেটে গেলেও মাঝপথ থেকে ডেটা রিকভার করা যায়।
*   **উদাহরণ**: NetBIOS, RPC, Sockets API।

#### Layer 4: Transport Layer (ট্রান্সপোর্ট লেয়ার)
*   **কাজ**: এটি **Process-to-Process** ডেলিভারি নিশ্চিত করে। অর্থাৎ প্রেরক কম্পিউটারের নির্দিষ্ট একটি অ্যাপ্লিকেশন (যেমন: Chrome ব্রাউজার) থেকে প্রাপক কম্পিউটারের নির্দিষ্ট অ্যাপ্লিকেশনে (যেমন: Web Server) ডেটা পৌঁছানো।
*   **মূল ফিচারসমূহ**:
    *   **Port Numbers**: সোর্স ও ডেস্টিনেশন অ্যাপ্লিকেশন চিহ্নিত করা (যেমন: HTTP = Port 80, HTTPS = Port 443)।
    *   **Segmentation**: বড় ডেটাকে ছোট ছোট অংশে বিভক্ত করা যাকে **Segments** বলে।
    *   **Flow Control**: রিসিভারের প্রসেসিং ক্ষমতার সাথে সামঞ্জস্য রেখে ডেটা পাঠানো।
    *   **Congestion Control**: নেটওয়ার্কে জ্যাম এড়ানো।
    *   **Error Control**: প্যাকেট নষ্ট হলে পুনরায় পাঠানো (TCP-এর ক্ষেত্রে)।
*   **প্রধান প্রোটোকল**: TCP (Transmission Control Protocol) এবং UDP (User Datagram Protocol)।

#### Layer 3: Network Layer (নেটওয়ার্ক লেয়ার)
*   **কাজ**: এটি **Host-to-Host** ডেলিভারি নিশ্চিত করে। অর্থাৎ এক নেটওয়ার্কের কম্পিউটার থেকে অন্য নেটওয়ার্কের কম্পিউটারে ডেটা পাঠানোর সর্বোত্তম পথ (Routing) খুঁজে বের করে।
*   **মূল ফিচারসমূহ**:
    *   **Logical Addressing**: আইপি অ্যাড্রেসিং (IPv4/IPv6)।
    *   **Routing**: রাউটার ব্যবহার করে প্যাকেটের জন্য সঠিক পথ নির্ধারণ করা।
*   **ডেটা ইউনিট**: **Packets** (প্যাকেট)।
*   **প্রধান প্রোটোকল**: IP (IPv4/IPv6), ICMP (Ping-এর জন্য), ARP (আইপি থেকে ম্যাক বের করার জন্য)।

#### Layer 2: Data Link Layer (ডাটা লিংক লেয়ার)
*   **কাজ**: একই লোকাল নেটওয়ার্কে (LAN) সরাসরি সংযুক্ত দুটি ডিভাইসের মধ্যে বিশ্বস্ত ডেটা ট্রান্সমিশন (Hop-to-Hop) নিশ্চিত করে। 
*   **মূল ফিচারসমূহ**:
    *   **Physical Addressing**: ম্যাক অ্যাড্রেস (MAC Address) ব্যবহার করে ফ্রেম ডেলিভারি।
    *   **Framing**: প্যাকেটগুলোকে ফ্রেমে রূপান্তর করা এবং শুরুতে ও শেষে হেডার-টেইলার যুক্ত করা।
    *   **Error Detection**: ট্রেইলারে থাকা CRC (Cyclic Redundancy Check) কোড দিয়ে ফ্রেমটি নষ্ট হয়েছে কিনা তা চেক করা।
*   **সাব-লেয়ার**:
    1.  **LLC (Logical Link Control)**: নেটওয়ার্ক লেয়ারের সাথে যোগাযোগ ও ফ্লো কন্ট্রোল করে।
    2.  **MAC (Media Access Control)**: ফিজিক্যাল মিডিয়াতে ডেটা পাঠানোর অনুমতি ও ফিজিক্যাল অ্যাড্রেসিং নিয়ন্ত্রণ করে।
*   **ডেটা ইউনিট**: **Frames** (ফ্রেম)।
*   **হার্ডওয়্যার**: Switch (সুইচ), Bridge, NIC।

#### Layer 1: Physical Layer (ফিজিক্যাল লেয়ার)
*   **কাজ**: ফিজিক্যাল মাধ্যম বা ক্যাবলের মধ্য দিয়ে বাইনারি বিটগুলো (`0` এবং `1`) বৈদ্যুতিক সংকেত, আলোক সংকেত বা রেডিও তরঙ্গে রূপান্তর করে পাঠিয়ে দেওয়া।
*   **মূল ফিচারসমূহ**: ক্যাবলের ধরন (ফাইবার, টুইস্টেড পেয়ার), কানেক্টর টাইপ (RJ45), ভোল্টেজ লেভেল, বিট রেট এবং নেটওয়ার্ক টপোলজি।
*   **ডেটা ইউনিট**: **Bits** (বিট)।
*   **হার্ডওয়্যার**: Hub (হাব), Repeater, Cables, Connectors.

## ৩. TCP/IP Reference Model (৪টি স্তর)
OSI একটি তাত্ত্বিক মডেল হলেও বাস্তবে ইন্টারনেটের সকল কাজ চলে **TCP/IP Model** বা **Internet Protocol Suite**-এর মাধ্যমে। এটি মূলত ৪টি লেয়ার বিশিষ্ট:

```
  OSI Layers (7)                    TCP/IP Layers (4)
┌─────────────────────────┐        ┌─────────────────────────┐
│ 7. Application          │        │                         │
├─────────────────────────┤        │ 4. Application          │
│ 6. Presentation         │ ------ │                         │
├─────────────────────────┤        │ (HTTP, DNS, SMTP)       │
│ 5. Session              │        │                         │
├─────────────────────────┴        ├─────────────────────────┤
│ 4. Transport            -------- │ 3. Transport (TCP, UDP) │
├─────────────────────────┬        ├─────────────────────────┤
│ 3. Network              -------- │ 2. Internet (IP)        │
├─────────────────────────┼        ├─────────────────────────┤
│ 2. Data Link            │        │ 1. Network Access       │
├─────────────────────────┤ ------ │                         │
│ 1. Physical             │        │ (Ethernet, Wi-Fi)       │
└─────────────────────────┘        └─────────────────────────┘
```

## ৪. Data Encapsulation & Decapsulation

যখন আপনি ব্রাউজারে `google.com` লিখে এন্টার চাপেন, তখন ডেটা কিভাবে নেটওয়ার্কের মাধ্যমে পরিবাহিত হয়? এই প্রক্রিয়াই হলো **Encapsulation** (ডেটা প্যাক করা) এবং **Decapsulation** (ডেটা আনপ্যাক করা)।

### Encapsulation (সেন্ডার সাইডে):
১. **Application Layer**: ইউজার একটি রিকোয়েস্ট তৈরি করে (যেমন: HTTP GET)। একে বলা হয় **Data**।
২. **Transport Layer**: ডেটার সাথে সোর্স ও ডেস্টিনেশন পোর্ট নম্বরসহ TCP হেডার যুক্ত করা হয়। একে বলা হয় **Segment**।
৩. **Network Layer**: সেগমেন্টের সাথে সোর্স ও ডেস্টিনেশন আইপি যুক্ত করা হয়। একে বলা হয় **Packet**।
৪. **Data Link Layer**: প্যাকেটের সাথে সোর্স ও ডেস্টিনেশন ম্যাক অ্যাড্রেসযুক্ত হেডার এবং ভুল সনাক্তকরণের ট্রেইলার যুক্ত করা হয়। একে বলা হয় **Frame**।
৫. **Physical Layer**: ফ্রেমটিকে বাইনারি বিটে (`0` ও `1`) রূপান্তর করে তারের মাধ্যমে পাঠিয়ে দেওয়া হয়।

```
[ Data ]                                    (Application Layer)
      │
      ▼
[ TCP Header | Data ]                       (Transport Layer - SEGMENT)
      │
      ▼
[ IP Header | TCP Header | Data ]            (Network Layer - PACKET)
      │
      ▼
[ MAC Header | IP Header | TCP | Data | CRC ] (Data Link Layer - FRAME)
      │
      ▼
 010100101010001101011010100110100100101     (Physical Layer - BITS)
```

### Decapsulation (রিসিভার সাইডে):
রিসিভার যখন বিটগুলো পায়, তখন সে উল্টোভাবে কাজ করে:
১. ফিজিক্যাল লেয়ার থেকে বিট সংগ্রহ করে ফ্রেমে রূপান্তর করে।
২. ডাটা লিংক লেয়ারে ম্যাক অ্যাড্রেস চেক করে হেডার ও টেইলার বাদ দিয়ে প্যাকেটটি নেটওয়ার্ক লেয়ারে পাঠায়।
৩. নেটওয়ার্ক লেয়ারে আইপি চেক করে আইপি হেডার বাদ দিয়ে সেগমেন্টটি ট্রান্সপোর্ট লেয়ারে পাঠায়।
৪. ট্রান্সপোর্ট লেয়ারে পোর্ট নম্বর দেখে কোন অ্যাপ্লিকেশন পাবে তা নিশ্চিত করে হেডার বাদ দিয়ে ডেটা অ্যাপ্লিকেশন লেয়ারে পাঠায়।
৫. অ্যাপ্লিকেশন লেয়ারে আসল রিকোয়েস্ট বা ডেটা পৌঁছায়।
