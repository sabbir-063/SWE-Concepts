# Chapter 1: Internet Protocol (IP)

স্বাগতম! কম্পিউটার নেটওয়ার্কিংয়ের সবচেয়ে গুরুত্বপূর্ণ ভিত্তি—**Internet Protocol (IP)** দিয়ে আমাদের যাত্রা শুরু হচ্ছে। 

এই গাইডটি এমনভাবে ডিজাইন করা হয়েছে যাতে আপনি প্রতিটি ধারণার গভীরতম স্তর বুঝতে পারেন। আমরা IPv4, IPv6, Subnetting, CIDR, NAT এবং DHCP নিয়ে বাস্তব উদাহরণসহ আলোচনা করব।

---

## ১. Introduction to IP (ইন্টারনেট প্রোটোকল কি?)
**Internet Protocol (IP)** হলো OSI মডেলের **Network Layer (Layer 3)**-এ ব্যবহৃত একটি প্রধান প্রোটোকল। এর মূল কাজ হলো এক ডিভাইস থেকে অন্য ডিভাইসে ডেটা (যাকে **Packets** বলা হয়) পাঠানো এবং রিসিভ করা।

### key characteristics of IP:
1. **Connectionless**: ডেটা পাঠানোর আগে সেন্ডার ও রিসিভারের মধ্যে কোনো পূর্ব সংযোগ (handshake) তৈরি হয় না। (এটি নিশ্চিত করার দায়িত্ব TCP-এর)।
2. **Best-Effort Delivery (Unreliable)**: IP গ্যারান্টি দেয় না যে সমস্ত প্যাকেট নিরাপদে এবং সঠিক ক্রমে পৌঁছাবে। প্যাকেট লস হতে পারে, ডুপ্লিকেট হতে পারে বা এলোমেলো হতে পারে।
3. **Media Independent**: ডেটা অপটিক্যাল ফাইবার, কপার ক্যাবল বা ওয়্যারলেস (Wi-Fi)—যেকোনো মাধ্যম দিয়েই যাক না কেন, IP সবখানেই সমানভাবে কাজ করে।

## ২. IPv4 vs IPv6 (আইপি সংস্করণসমূহ)

বর্তমানে আমরা দুটি আইপি সংস্করণ ব্যবহার করি: **IPv4** এবং **IPv6**।

```
+------------------+------------------------------------+----------------------------------------+
| Feature          | IPv4                               | IPv6                                   |
+------------------+------------------------------------+----------------------------------------+
| Address Size     | 32-bit (4 Bytes)                   | 128-bit (16 Bytes)                     |
| Address Space    | 2^32 ≈ 4.3 Billion addresses       | 2^128 (প্রায় অসীম, 340 undecillion)    |
| Format           | Dotted Decimal (e.g., 192.168.1.1) | Hexadecimal Colon (e.g., 2001:db8::1)  |
| Configuration    | Manual or DHCP                     | Stateless Address Autoconfig (SLAAC)   |
| Security         | Optional IPSec                     | Built-in (Mandatory IPSec support)     |
| Header Size      | Variable (20 to 60 bytes)          | Fixed (40 bytes)                       |
+------------------+------------------------------------+----------------------------------------+
```

### IPv4 Class-Based Addressing (ক্লাসভিত্তিক আইপি)
অতীতে IPv4-কে পাঁচটি ক্লাসে ভাগ করা হয়েছিল:
- **Class A**: `1.0.0.0` to `127.255.255.255` (বিশাল নেটওয়ার্কের জন্য, যেমন বড় কর্পোরেশন বা ISP)
- **Class B**: `128.0.0.0` to `191.255.255.255` (মাঝারি আকারের নেটওয়ার্ক)
- **Class C**: `192.0.0.0` to `223.255.255.255` (ছোট নেটওয়ার্ক, যেমন আমাদের হোম রাউটার)
- **Class D**: `224.0.0.0` to `239.255.255.255` (Multicasting বা গ্রুপ যোগাযোগের জন্য সংরক্ষিত)
- **Class E**: `240.0.0.0` to `255.255.255.255` (গবেষণা ও পরীক্ষার জন্য সংরক্ষিত)

> [!NOTE]
> **Loopback Address**: `127.0.0.1` হলো লুপব্যাক অ্যাড্রেস। এটি আপনার নিজের কম্পিউটারকে (localhost) নির্দেশ করে এবং নেটওয়ার্ক কার্ড ঠিকমতো কাজ করছে কিনা তা টেস্ট করতে ব্যবহৃত হয়।

## ৩. Subnetting & CIDR (সাবনেটিং ও সিআইডিআর)

ধরা যাক, একটি কোম্পানিকে Class C-এর একটি সম্পূর্ণ আইপি ব্লক দেওয়া হলো (যেমন `192.168.1.0/24`)। এতে ২৫৪টি আইপি এড্রেস থাকবে। কিন্তু কোম্পানির মাত্র ৩টি ছোট ডিপার্টমেন্ট আছে যাদের প্রতিটি বিভাগে মাত্র ২০টি করে পিসি রয়েছে। আমরা যদি পুরো ব্লকটি একটি বিভাগকে দিয়ে দিই, তবে বাকি আইপিগুলো নষ্ট হবে। এই অপচয় রোধ করতে একটি বড় নেটওয়ার্ককে ছোট ছোট সাব-নেটওয়ার্কে ভাগ করার প্রক্রিয়াকেই **Subnetting** বলে।

### CIDR Notation (Classless Inter-Domain Routing)
CIDR-এ আইপি এড্রেসের সাথে একটি স্ল্যাশ (`/`) দিয়ে নেটওয়ার্ক বিটের সংখ্যা প্রকাশ করা হয়। একে **Slash Notation** বা **CIDR Mask** বলে।
যেমন: `192.168.1.10/24`
এখানে `/24` মানে হলো প্রথম ২৪টি বিট হচ্ছে **Network ID** এবং বাকি ৮টি বিট (32 - 24) হচ্ছে **Host ID**।

### Subnetting Calculation Example (হাতে কলমে সাবনেটিং)
চলুন আমরা `192.168.1.0/26` সাবনেটটির হিসাব করি:

1. **Subnet Mask বের করা**:
   - `/26` মানে প্রথম ২৬টি বিট ১ (ON) এবং বাকি ৬টি বিট ০ (OFF)।
   - Binary: `11111111.11111111.11111111.11000000`
   - Decimal: `255.255.255.192`

2. **Total IPs in this Subnet**:
   - সূত্র: $2^{\text{Host Bits}} = 2^6 = 64$ টি আইপি এড্রেস।

3. **Usable IPs**:
   - প্রতিটি সাবনেটে প্রথম এবং শেষ আইপি ব্যবহার করা যায় না।
     - প্রথম আইপি = **Network Address** (নেটওয়ার্ককে রিপ্রেজেন্ট করে)।
     - শেষ আইপি = **Broadcast Address** (ঐ নেটওয়ার্কের সবাইকে মেসেজ পাঠাতে ব্যবহৃত হয়)।
   - সূত্র: $2^{\text{Host Bits}} - 2 = 64 - 2 = 62$ টি ব্যবহারযোগ্য আইপি।

4. **Range of IP addresses**:
   - **Network Address**: `192.168.1.0`
   - **First Usable IP**: `192.168.1.1`
   - **Last Usable IP**: `192.168.1.62`
   - **Broadcast Address**: `192.168.1.63`

## ৪. NAT (Network Address Translation)

পৃথিবীতে মোট IPv4 এড্রেস আছে ৪.৩ বিলিয়ন, যা বর্তমান জনসংখ্যার তুলনায় অনেক কম। এই তীব্র আইপি সংকট দূর করতে **NAT** উদ্ভাবন করা হয়।

### RFC 1918 Private IP Ranges (প্রাইভেট আইপি)
প্রাইভেট আইপি ইন্টারনেটে সরাসরি রাউট বা চলাচল করতে পারে না। এগুলো শুধুমাত্র লোকাল নেটওয়ার্কে (যেমন আপনার বাসা বা অফিস) ব্যবহৃত হয়।
- **Class A**: `10.0.0.0` to `10.255.255.255`
- **Class B**: `172.16.0.0` to `172.31.255.255`
- **Class C**: `192.168.0.0` to `192.168.255.255`

### NAT কিভাবে কাজ করে?
লোকাল নেটওয়ার্কের প্রাইভেট আইপিকে পাবলিক আইপিতে রূপান্তর করে ইন্টারনেটে অ্যাক্সেস দেওয়ার কাজ করে **NAT** (সাধারণত আমাদের ঘরের রাউটারে এই কাজ হয়)।

```
  [ Local PC ] (Private IP: 192.168.1.50)
        │
        ▼ (Sends packet to local router)
  [ Router / NAT Gateway ] (Public IP: 203.0.113.10)
        │
        ▼ (Translates source IP to 203.0.113.10 and assigns unique source port, e.g., 50001)
  [ Internet / Google Server ]
```

### Types of NAT:
1. **Static NAT (1:1)**: একটি নির্দিষ্ট প্রাইভেট আইপির বিপরীতে একটি নির্দিষ্ট পাবলিক আইপি বরাদ্দ করা হয় (সাধারণত ওয়েব সার্ভারের জন্য ব্যবহৃত হয়)।
2. **Dynamic NAT**: একগুচ্ছ পাবলিক আইপি থাকে। লোকাল হোস্ট যখন ইন্টারনেট অ্যাক্সেস করতে চায়, তখন ঐ গুচ্ছ থেকে খালি থাকা একটি আইপি তাকে সাময়িকভাবে দেওয়া হয়।
3. **PAT (Port Address Translation / NAT Overload)**: এটি সবচেয়ে বেশি ব্যবহৃত হয়। এখানে একটিমাত্র পাবলিক আইপির বিভিন্ন **Port Number** ব্যবহার করে লোকাল নেটওয়ার্কের হাজার হাজার পিসিকে ইন্টারনেট সংযোগ দেওয়া হয়।

## ৫. DHCP (Dynamic Host Configuration Protocol)

আপনি যখনই কোনো নতুন ওয়াইফাই বা ল্যান সংযোগে যুক্ত হন, আপনার ডিভাইস স্বয়ংক্রিয়ভাবে একটি আইপি এড্রেস, সাবনেট মাস্ক, গেটওয়ে ও ডিএনএস পেয়ে যায়। এই কাজটি করে **DHCP**।

DHCP কিভাবে কাজ করে? এটি **DORA Process** মেনে চলে:

```
Client                                     DHCP Server
  │                                             │
  ├─────── DHCP Discover (Broadcast) ──────────>│ (পিসি নেটওয়ার্কে চিৎকার করে জিজ্ঞেস করে, "কেউ কি সার্ভার আছেন?")
  │                                             │
  |<────── DHCP Offer (Unicast/Broadcast) ──────┤ (সার্বার বলে, "হ্যাঁ, আমি আছি! তুমি এই আইপিটি নিতে পারো।")
  │                                             │
  ├─────── DHCP Request (Broadcast) ───────────>│ (পিসি বলে, "ধন্যবাদ, আমি এই আইপিটাই নিতে চাই। লক করে দিন।")
  │                                             │
  |<────── DHCP Ack (Unicast/Broadcast) ────────┤ (সার্বার বলে, "ঠিক আছে, আইপিটি তোমার জন্য বরাদ্দ করা হলো!")
  │                                             │
```

## ৬. IP Routing & Packet Forwarding

যখন কোনো কম্পিউটার ইন্টারনেটে একটি প্যাকেট পাঠায়, রাউটারগুলো তার **Routing Table** দেখে সিদ্ধান্ত নেয় প্যাকেটটি কোন পথে যাবে। 

### Routing Table Elements:
- **Destination Network**: প্যাকেটটি যেখানে যেতে চায় তার নেটওয়ার্ক রেঞ্জ (যেমন `104.244.42.0/24`)।
- **Subnet Mask**: নেটওয়ার্ক আইডি চেনার জন্য মাস্ক।
- **Next Hop / Gateway**: প্যাকেটটি পরবর্তী কোন রাউটারে পাঠাতে হবে।
- **Interface**: রাউটারের কোন পোর্ট বা কেবল দিয়ে প্যাকেটটি বের হবে।
- **Default Route (`0.0.0.0/0`)**: যদি ডেস্টিনেশন এড্রেসের জন্য রাউটিং টেবিলে কোনো নির্দিষ্ট এন্ট্রি না থাকে, তবে রাউটার এই ডিফল্ট গেটওয়ে বা আপস্ট্রিম প্রোভাইডারের দিকে প্যাকেটটি পাঠিয়ে দেয়।
